\section{Variables objetivo}
Para brindar un contexto adecuado, es necesario describir las variables objetivo (\textit{targets}) presentes en el conjunto de datos. Como se mencionó previamente, el modelo propuesto tiene como finalidad predecir las siguientes variables:

\begin{itemize}
    \item \textbf{Vegetación verde seca -- Dry\_Green\_g}: corresponde a la vegetación verde distinta del trébol, medida en gramos.
    
    \item \textbf{Material seco muerto -- Dry\_Dead\_g}: representa el material senescente presente en la muestra, medido en gramos.
    
    \item \textbf{Biomasa seca de trébol -- Dry\_Clover\_g}: corresponde a la biomasa asociada al componente de trébol, medida en gramos.
    
    \item \textbf{Materia seca verde -- GDM\_g}: se define como la suma de la vegetación verde y el trébol, expresada en gramos.
    
    \item \textbf{Biomasa seca total -- Dry\_Total\_g}: representa la biomasa total, resultante de la combinación de todos los componentes anteriores, medida en gramos. Esta variable es la más relevante para el presente proyecto.
\end{itemize}

Cada una de estas variables se encuentra expresada en gramos y conforma el conjunto de valores objetivo que deben ser predichos por los modelos desarrollados. En conjunto, estas cinco variables permiten caracterizar de manera detallada la composición de la biomasa presente en las muestras analizadas.

A continuación, se presentan las métricas de evaluación utilizadas para analizar el desempeño de los modelos implementados.

\section{Métricas de evaluación}
Las métricas de evaluación utilizadas en el presente proyecto fueron el error cuadrático medio (MSE), la raíz del error cuadrático medio (RMSE) y el coeficiente de determinación ($R^2$). Estas métricas se calcularon utilizando la librería \textit{scikit-learn} \cite{scikit-learn}, la cual proporciona implementaciones estándar para su evaluación.

Adicionalmente, para la competencia de Kaggle \cite{csiro-biomass}, se emplea una variante del coeficiente de determinación, denominada $R^2$ ponderado, la cual considera todas las predicciones de manera conjunta mediante un esquema de pesos asociado a cada variable objetivo.

En este enfoque, cada fila del conjunto de datos se pondera de acuerdo con el tipo de variable objetivo, utilizando los siguientes pesos:

\begin{itemize}
    \item Dry\_Green\_g: 0.1
    \item Dry\_Dead\_g: 0.1
    \item Dry\_Clover\_g: 0.1
    \item GDM\_g: 0.2
    \item Dry\_Total\_g: 0.5
\end{itemize}

Esto implica que, en lugar de calcular el $R^2$ de forma independiente para cada variable y posteriormente promediar los resultados, se calcula un único $R^2$ global ponderado considerando todas las observaciones en conjunto.

El coeficiente de determinación ponderado se define de la siguiente manera:

\begin{equation}
R^2_w = 1 - \frac{\sum_{j} w_j (y_j - \hat{y}_j)^2}{\sum_{j} w_j (y_j - \bar{y}_w)^2}
\end{equation}

donde la media ponderada global $\bar{y}_w$ se calcula como:

\begin{equation}
\bar{y}_w = \frac{\sum_{j} w_j y_j}{\sum_{j} w_j}
\end{equation}

En estas expresiones:

\begin{itemize}
    \item $y_j$: valor real (\textit{ground truth}) para la observación $j$
    \item $\hat{y}_j$: valor predicho por el modelo para la observación $j$
    \item $w_j$: peso por fila basado en el tipo de variable objetivo
    \item $\bar{y}_w$: media ponderada global de los valores reales
\end{itemize}

Este esquema de evaluación permite asignar mayor importancia a variables clave como \textit{Dry\_Total\_g}, la cual tiene un peso superior, alineando así la métrica con los objetivos principales del problema. A continuación, se presentan los resultados obtenidos por cada una de las soluciones propuestas, junto con la explicación de su desempeño en función de las métricas de evaluación utilizadas. Asimismo, se analizan los modelos, destacando el comportamiento observado en la predicción de las distintas variables objetivo.



\section{Resultados de la Solución 1: VGG16 + Bosque Aleatorio}
La solución se basa en el uso de la arquitectura VGG16, disponible en la biblioteca Keras dentro del entorno TensorFlow, como módulo de extracción de características. Este modelo permite transformar las imágenes en vectores numéricos que capturan información relevante sobre la estructura visual del pastizal. Posteriormente, estos vectores son utilizados como entrada para un modelo de Bosque Aleatorio, el cual se encarga de predecir las variables objetivo relacionadas con la biomasa.

La elección de esta combinación responde a la necesidad de contar con una solución robusta ante el ruido y capaz de modelar relaciones no lineales entre las características visuales extraídas y los valores de biomasa. Además, el uso de Bosque Aleatorio permite reducir el riesgo de sobreajuste al combinar múltiples árboles de decisión.

En términos de implementación, se utilizó un tamaño de imagen de $224 \times 224$ píxeles, acorde con los requerimientos de VGG16. Se aplicó un conjunto básico de aumento de datos, incluyendo rotaciones, volteos y ajustes de brillo, contraste, con el fin de mejorar la capacidad de generalización del modelo. Asimismo, el conjunto de datos se dividió en un $80\%$ para entrenamiento y un $20\%$ para validación.

Para el modelo de regresión, se empleó un Bosque Aleatorio con 300 árboles (\textit{n\_estimators = 300}) y una semilla fija (\textit{random\_state = 999}) para garantizar la reproducibilidad de los resultados. Es importante destacar que los pesos del modelo VGG16 se mantuvieron congelados durante todo el proceso de entrenamiento, lo que significa que sus parámetros internos no fueron modificados. En este enfoque, el modelo se utiliza únicamente como un extractor de características, aprovechando el conocimiento previamente aprendido a partir de grandes conjuntos de imágenes. De esta manera, VGG16 transforma cada imagen en un vector numérico que resume sus patrones visuales más relevantes, el cual es posteriormente utilizado por el modelo de regresión. Esta estrategia permite reducir el costo computacional y evitar problemas de sobreajuste, especialmente considerando el tamaño limitado del conjunto de datos disponible.

\subsubsection{Análisis de resultados}

En términos generales, el modelo presenta un desempeño moderado. El valor promedio de RMSE es aproximadamente de $16$ g/m², lo que indica que, en promedio, las predicciones del modelo difieren del valor real en alrededor de 16 gramos por muestra. Esto proporciona una medida intuitiva del error del sistema.

Es importante destacar que los valores de $R^2$ obtenidos en la validación local y en la plataforma Kaggle no son directamente comparables, ya que se calculan bajo condiciones distintas. En el caso de la validación local, el $R^2$ se obtiene utilizando únicamente el $20\%$ del conjunto de datos reservado para validación, evaluando cada variable objetivo por separado y posteriormente promediando los resultados. Por otro lado, en Kaggle el $R^2$ se calcula sobre un conjunto de datos privado no visible durante el entrenamiento, lo que permite medir la capacidad real de generalización del modelo.

Adicionalmente, Kaggle utiliza una variante del $R^2$ ponderado, en la cual todas las observaciones se consideran en conjunto y se asignan distintos pesos a cada variable objetivo, dando mayor importancia a variables como \textit{Dry\_Total\_g}. Esto introduce una diferencia adicional respecto al cálculo tradicional del $R^2$, lo que explica por qué el valor obtenido en Kaggle ($0.42455$) es superior al promedio local ($0.2806$).

El valor de $R^2 = 0.42455$ obtenido en Kaggle indica que el modelo es capaz de explicar aproximadamente un $42\%$ de la variabilidad total de los datos en el conjunto de evaluación. En otras palabras, aunque el modelo logra capturar una parte significativa de los patrones presentes en las imágenes, aún existe una proporción considerable de la variabilidad que no es explicada, lo cual refleja la complejidad del problema.

Uno de los principales factores que influyen en estos resultados es el tamaño limitado del conjunto de datos. Aunque el dataset original contiene más de mil imágenes, la versión disponible para entrenamiento en la competencia cuenta únicamente con aproximadamente 300 muestras. Esta cantidad resulta relativamente pequeña para entrenar modelos de aprendizaje automático, especialmente en problemas de visión por computador.

Si bien se aplicaron técnicas de aumento de datos para mitigar este problema, su uso debe ser moderado, ya que un exceso de transformaciones puede provocar sobreajuste, es decir, que el modelo aprenda patrones artificiales que no generalizan bien a nuevos datos.

Por otra parte, la naturaleza del problema también introduce dificultad adicional debido a que se busca estimar biomasa a partir de imágenes en dos dimensiones, lo cual implica inferir información tridimensional (cantidad de material vegetal) a partir de una representación visual plana.

Asimismo, se observa que variables como \textit{Dry\_Clover\_g} presentan un mejor desempeño relativo, mientras que variables más complejas como \textit{Dry\_Total\_g} presentan mayor error. Esto se debe a que esta última integra múltiples componentes, lo que aumenta la complejidad del problema.

En conclusión, esta primera solución cumple su objetivo como línea base, permitiendo identificar las principales limitaciones del problema y sentando las bases para el desarrollo de enfoques más avanzados en las siguientes iteraciones.


\section{Solución 2: EfficientNet + Ridge Regression}
Como alternativa a la solución anterior, se propone el uso de una arquitectura más moderna para la extracción de características, específicamente EfficientNet, disponible en la biblioteca Keras dentro del entorno TensorFlow. Posteriormente, para la etapa de estimación, se emplea un modelo de Regresión Ridge, el cual introduce un término de regularización con el objetivo de controlar el sobreajuste.

La motivación de esta configuración radica en evaluar si un extractor de características más avanzado, combinado con un modelo lineal regularizado, puede ofrecer una mayor estabilidad en las predicciones. A diferencia de la solución anterior, en este enfoque \textbf{no se utilizó aumento de datos}, ya que experimentalmente no contribuyó a mejorar el desempeño del modelo.

En términos de implementación, EfficientNet se utilizó como extractor de características con pesos preentrenados congelados, generando un vector de características de dimensión 1536 por imagen. Posteriormente, estos vectores fueron normalizados mediante \textit{StandardScaler} antes de ser utilizados por el modelo de Regresión Ridge. 

Se evaluaron distintos valores de $\alpha \in \{0.1, 1.0, 10, 100, 1000\}$ con el objetivo de identificar la configuración más adecuada, seleccionando finalmente $\alpha = 1000$ al ser el que produjo el mejor desempeño en el conjunto de validación. Este parámetro controla la fuerza de la regularización en el modelo de Regresión Ridge, en términos simples, determina qué tanto se penalizan los coeficientes grandes. Valores altos de $\alpha$ obligan al modelo a mantener coeficientes pequeños, lo que da lugar a una solución más simple y menos propensa al sobreajuste. En este caso, la elección de un valor elevado resulta coherente con la alta dimensionalidad del vector de características y la limitada cantidad de muestras disponibles, ya que una regularización fuerte contribuye a estabilizar las estimaciones y mejorar la capacidad de generalización del modelo.

\subsubsection{Análisis de resultados}

En comparación con la Solución 1, los resultados obtenidos son ligeramente inferiores en términos del puntaje en Kaggle, aunque se mantienen en un rango similar. El RMSE promedio es de aproximadamente $15.58$ g/m², lo cual indica que el error promedio del modelo es cercano al observado en la solución anterior, mostrando una consistencia en el nivel de precisión alcanzado.

La principal diferencia radica en el uso de un modelo lineal regularizado (Ridge), el cual, aunque es más estable y menos propenso al sobreajuste, también posee una menor capacidad para capturar relaciones complejas y no lineales presentes en los datos. Esto puede explicar por qué, a pesar de utilizar un extractor de características más moderno como EfficientNet, el desempeño global no supera significativamente al de la solución basada en Bosque Aleatorio.

Asimismo, la decisión de no utilizar aumento de datos, aunque justificada para evitar problemas de generalización en modelos lineales, limita la variabilidad del conjunto de entrenamiento, lo que puede influir en la capacidad del modelo para adaptarse a nuevas imágenes.

El valor de $R^2 = 0.36862$ obtenido en Kaggle indica que el modelo logra explicar aproximadamente un $36\%$ de la variabilidad de los datos en el conjunto de evaluación, lo cual es ligeramente inferior al obtenido en la solución anterior, pero aún refleja un desempeño razonable considerando las limitaciones del problema. 

En conclusión, esta segunda solución demuestra que el uso de modelos lineales regularizados ofrece una alternativa más estable, aunque con una ligera pérdida en capacidad predictiva. Esto se debe a que la relación entre las características extraídas de las imágenes y la biomasa no es estrictamente lineal. En otras palabras, cambios pequeños en la imagen (como variaciones en textura, densidad o iluminación) no se traducen de manera proporcional en cambios en la cantidad de biomasa.

Además, la biomasa es una propiedad física compleja que depende de múltiples factores visuales que interactúan entre sí, como la superposición de hojas, la presencia de sombras o la estructura tridimensional de la vegetación, elementos que no pueden ser capturados completamente en una representación bidimensional. Por ejemplo, dos imágenes pueden verse similares pero contener diferentes cantidades reales de biomasa debido a efectos de oclusión o perspectiva.

Un modelo lineal, como Ridge, solo puede aprender relaciones proporcionales simples entre las variables, por lo que no logra capturar estas interacciones complejas. Esto limita su capacidad para modelar con precisión el problema, lo que explica la ligera disminución en el desempeño.

El resultado en Kaggle ($R^2 = -0.43519$) confirma este comportamiento, evidenciando que el modelo no logra generalizar a datos nuevos y que su capacidad predictiva es muy limitada.

A diferencia de las soluciones anteriores, este enfoque no aporta ventajas significativas ni una base sólida sobre la cual construir modelos más avanzados, debido a sus limitaciones inherentes para modelar relaciones complejas en datos de tipo visual.

\subsubsection{Análisis de resultados – Solución 4}
Uno de los principales factores que puede explicar este comportamiento es la naturaleza del modelo SVR. Este tipo de modelo es altamente sensible a la selección de hiperparámetros como $C$, $\epsilon$ y el kernel. En este caso, se utilizaron valores por defecto, lo cual limita su capacidad para adaptarse adecuadamente a la complejidad del problema. Sin una optimización adecuada (por ejemplo, mediante búsqueda en rejilla o validación cruzada), el modelo puede quedar subajustado o no capturar correctamente las relaciones en los datos.

Otro aspecto importante es la alta dimensionalidad del espacio de características (2048 dimensiones) en comparación con el número de muestras disponibles. Este desbalance dificulta el aprendizaje del modelo, especialmente en algoritmos como SVR, que no escalan eficientemente en espacios de alta dimensión sin una adecuada reducción o selección de características.

Asimismo, al igual que en soluciones anteriores, el extractor de características se mantiene congelado. Esto implica que las representaciones generadas no están optimizadas específicamente para la tarea de estimación de biomasa, lo que limita el potencial del modelo, incluso cuando se utiliza un método no lineal como SVR.

En conclusión, aunque esta solución no alcanza un rendimiento competitivo, no se debe descartar completamente el uso de SVR. Con una mejor selección de hiperparámetros, posibles técnicas de reducción de dimensionalidad y un ajuste más fino del diseño, este enfoque podría mejorar su desempeño. Sin embargo, en su configuración actual, no logra aprovechar plenamente las ventajas teóricas del modelo.

En resumen, la Solución 1 es la que obtuvo el mejor desempeño entre las alternativas evaluadas, por lo que se toma como referencia para el análisis posterior. A continuación, se presenta un estudio del error de predicción de biomasa utilizando imágenes del conjunto de datos, calculado como $|\text{biomasa real} - \text{biomasa predicha}|$. Este análisis permite visualizar de manera más intuitiva el comportamiento del modelo, observando directamente sus salidas y cómo varían respecto a los valores reales.

\section{Parámetros para la biomasa}

En el Capítulo 3 se discutieron de manera detallada los parámetros relevantes para la estimación de biomasa. En esta sección se presentan únicamente como un resumen de las variables que deberían considerarse al construir conjuntos de datos similares orientados al análisis de biomasa en malezas.

Aunque estos parámetros no fueron utilizados directamente en la solución desarrollada, se consideran esenciales durante la etapa de recolección y estructuración de datos, debido a que aportan información relevante sobre la vegetación. Por lo tanto estos parámetros deben registrarse dentro de futuros conjuntos de datos relacionados con biomasa en malezas.

Los principales parámetros identificados fueron:

\begin{itemize}
    \item \textbf{NDVI:} Permite estimar la densidad y el estado de salud de la vegetación.
    
    \item \textbf{Altura de la vegetación:} Relacionada con la cantidad de biomasa presente en el área de estudio.
    
    \item \textbf{Especie de la vegetación:} Permite diferenciar patrones de crecimiento y acumulación de biomasa entre distintas malezas.
    
    \item \textbf{Biomasa:} Variable objetivo utilizada como referencia para el entrenamiento de modelos de aprendizaje automático.
\end{itemize}
